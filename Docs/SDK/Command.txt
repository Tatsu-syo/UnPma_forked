Usage:
UnPma32.dll [command] [<-|/>switches...] <arcfile> [dir\] [file...|@filelist]
現在はコマンドを省略することはできません。

command:
l				リスティング。
c(v)			リスティング(パス付き)。
e				解凍(パスなし)。
x				解凍(パス付き)。

s				SFXを付加します。
				s [sfxfile] <arcfile> <sfxarc>
u				SFXを取り除きます。
				u <sfxarc> <arcfile>

switches:
--?				コマンドライン解析を終了[書庫名の直前で--@]
-a				属性を解凍				[-a1]
-b				破損ファイルを残す		[-b0]
-e				警告をエラーとして扱う	[-e0]
-f				強制的に上書き			[-f0]
-n				ウィンドウを抑止		[-n0]
-p				全パスで合致			[-p0]
-t				時刻を解凍				[-t1]
-w				一時ディレクトリを指定	[-w.]

arcfile:
操作の対象となる書庫ファイルです。
sfxfile:
自己解凍モジュールです。指定されなかった場合は内蔵モジュールが使用されます。
sfxarc:
自己解凍書庫です。
dir:
解凍先です。
file:
処理の対象とするファイルを指定します。
filelist:
レスポンスファイルです。--?を指定しないと@が開始文字になります。

オプションの詳細
--? コマンドライン解析を終了します。?がレスポンスファイルの開始文字になります。

-a 属性を解凍します。
PMA.comとは初期値が逆ですのでご注意ください。

-b CRCエラーの際ファイルを削除しません。
一部分でも解凍したいという場合以外はお勧めしません。

-e 警告をエラーとして扱います。
ただし、現在はファイルオープンエラー、CRCエラー、メソッドエラーしか警告としていませんので注意してください。
これ以外のエラーは警告程度の軽微なものであったとしても自動的にエラーになります。

-f 強制的に上書きします。
0: 問い合わせる
1: 上書きする
2: 上書きしない
PMA.comとの互換性を考慮し、NOA32w.dllのoスイッチなどとでは0と2が逆になっています。

-n ウィンドウを抑止します。
対話型ウィンドウは抑止されません。
これも抑止するには-f1か-f2を併用してください。

-p 全パスで合致
*\a.txtと指定して、d1\a.txtだけでなくd1\d2\a.txtも解凍してしまうことがないようにします。
ただし、*が「ルートディレクトリに属する」全てのファイルという意味になってしまいますので、
使用の際は十分にご注意ください。

-t 時刻を解凍
格納されている時刻を復元します。
ただし、秒数はDOS時間なので2秒単位でしか復元できません。

-w 一時ファイルを作成するディレクトリを、wに続けて指定します。
現在はsコマンドでのみ使用されます。
存在しないディレクトリを指定すると、GetTempFileName()に失敗するためERROR_SHARINGが返ります。
